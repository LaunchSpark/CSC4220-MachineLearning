Machine Learning Assignment 6: Ensemble Methods - Bagging

Context and Motivation

The Problem of Model Uncertainty

In Assignment 5, we saw that regularization helps control overfitting by reducing model capacity. But there's a deeper issue: any single model has inherent uncertainty. Even with perfect cross-validation, your chosen model might simply be "unlucky" - it might learn noise patterns that happen to look like signal in your specific dataset, or miss signal patterns that are hard to detect.

Why This Matters

In production machine learning with moderate datasets (hundreds to thousands of samples):

  • Both overfitting and underfitting are real risks.
  • Any single model selection carries uncertainty.
  • We need strategies to hedge against poor model choices.

The Ensemble Solution

Instead of putting all our trust in one model, we can use Bagging:

Train multiple models and average them to cancel out the noise each individual model learns. This approach hedges against the risk of poor model selection by combining multiple models' perspectives.

Learning Objectives

By the end of this assignment, you will be able to:

  • Understand that trained models contain learned signal + learned noise, where noise doesn't generalize.
  • Recognize that any single model has uncertainty in its generalization performance.
  • Implement bagging to reduce overfitting by canceling noise through averaging.
  • Diagnose overfitting and apply bagging as a solution.
  • Connect ensemble methods to the fundamental signal/noise and overfitting/underfitting tradeoffs.

Part 1: Understanding Signal, Noise, and Model Uncertainty (10 points)

Problem 1.1: Models Learn Signal + Noise and the Capacity Tradeoff (10 points)

Conceptual Setup

Real data can be decomposed as:

$$ y = f_{\text{true}}(x) + \epsilon $$

Where:

  • $f_{\text{true}}(x)$: The true underlying pattern (generalizes to new data).
  • $\epsilon$: Noise - random variations specific to your training set.

When you train a model on limited data, it learns:

$$ \text{model}(x) = f_{\text{signal}}(x) + f_{\text{noise}}(x) $$

Where:

  • $f_{\text{signal}}$: Approximates $f_{\text{true}}$ (this part generalizes).
  • $f_{\text{noise}}$: Fits the noise $\epsilon$ in your specific training data (this part doesn't generalize).

The key insight: With moderate datasets, it's nearly impossible for a model to learn only signal. Some noise will always creep in.

Scenario

Suppose you train 3 models on bootstrap samples (random resamples with replacement) from the same dataset:

Problem 2.1: Bootstrap Sampling (10 points)

Bootstrap sampling creates different training sets by sampling with replacement. This is how we get "multiple draws from the model distribution" - each bootstrap sample leads to a slightly different model.

Key idea: With 180 samples, sampling 180 with replacement means:

  • Getting 180 random draws with replacement from the dataset.
  • $\approx 63\%$ (statistically) unique samples (some samples appear multiple times).
  • $\approx 37\%$ of samples never selected.
  • Different bootstrap samples $\rightarrow$ different models $\rightarrow$ different noise learned.

Implement this function:

def bootstrap_sample(X, y, random_seed=None):
    if random_seed is not None:
        np.random.seed(random_seed)
    n = len(X)
    indices = np.random.choice(n, size=n, replace=True)
    return X[indices], y[indices]

class BaggingRegressor:
    def __init__(self, base_model_class, n_models=20, random_seed=42):
        self.base_model_class = base_model_class
        self.n_models = n_models
        self.random_seed = random_seed
        self.models = []

    def fit(self, X, y, **kwargs):
        self.models = []
        for i in range(self.n_models):
            X_boot, y_boot = bootstrap_sample(X, y, random_seed=self.random_seed + i)
            model = self.base_model_class(**kwargs)
            model.fit(X_boot, y_boot)
            self.models.append(model)

    def predict(self, X):
        preds = self.predict_individual(X)
        return np.mean(preds, axis=0)

    def predict_individual(self, X):
        preds = []
        for model in self.models:
            preds.append(model.predict(X))
        return np.array(preds)

def cross_validate_lambda(X_train, y_train, lambda_values, k_folds=5, alpha=0.01, iterations=1000):
    n = len(X_train)
    fold_size = n // k_folds
    indices = np.arange(n)
    np.random.shuffle(indices)
    cv_results = {}
    best_lambda = None
    min_mse = float('inf')
    for lam in lambda_values:
        fold_mses = []
        for i in range(k_folds):
            val_idx = indices[i*fold_size : (i+1)*fold_size]
            train_idx = np.concatenate([indices[:i*fold_size], indices[(i+1)*fold_size:]])
            X_f_train, y_f_train = X_train[train_idx], y_train[train_idx]
            X_f_val, y_f_val = X_train[val_idx], y_train[val_idx]
            model = VectorizedMLR(alpha=alpha, iterations=iterations, lambda_reg=lam)
            model.fit(X_f_train, y_f_train)
            mse = mean_squared_error(y_f_val, model.predict(X_f_val))
            fold_mses.append(mse)
        avg_mse = np.mean(fold_mses)
        cv_results[lam] = avg_mse
        if avg_mse < min_mse:
            min_mse = avg_mse
            best_lambda = lam
    return best_lambda, cv_results

A.1: Three-Way Split Function

def three_way_split(X, y, train_size=0.6, val_size=0.2, random_seed=42):
    """
    Split data into train, validation, and test sets.

    The three-way split is critical for ensemble methods:
    - Train: Build individual models
    - Validation: Select ensemble hyperparameters (n_models)
    - Test: Evaluate final performance (simulates production data)

    Parameters:
    -----------
    X : numpy array (n, p)
    y : numpy array (n, 1)
    train_size : float (default 0.6)
    val_size : float (default 0.2)
    random_seed : int (default 42)

    Returns:
    --------
    X_train, y_train : Training set (60%)
    X_val, y_val : Validation set (20%)
    X_test, y_test : Test set (20%)
    """
    np.random.seed(random_seed)
    n = len(X)
    indices = np.random.permutation(n)

    train_end = int(train_size * n)
    val_end = train_end + int(val_size * n)

    train_idx = indices[:train_end]
    val_idx = indices[train_end:val_end]
    test_idx = indices[val_end:]

    return (X[train_idx], y[train_idx],
            X[val_idx], y[val_idx],
            X[test_idx], y[test_idx])

class VectorizedMLR:
    """
    Multiple Linear Regression using gradient descent.
    Assumes students have this from previous assignments.
    """
    def __init__(self, alpha=0.01, iterations=1000, lambda_reg=0.0):
        self.alpha = alpha
        self.iterations = iterations
        self.lambda_reg = lambda_reg
        self.w = None
        self.b = None

    def fit(self, X, y):
        n, p = X.shape
        self.w = np.zeros((p, 1))
        self.b = 0

        for _ in range(self.iterations):
            y_pred = X @ self.w + self.b
            error = y_pred - y

            # Gradients
            dw = (2/n) * (X.T @ error) + (2 * self.lambda_reg * self.w)
            db = (2/n) * np.sum(error)

            # Update
            self.w -= self.alpha * dw
            self.b -= self.alpha * db

    def predict(self, X):
        return X @ self.w + self.b
